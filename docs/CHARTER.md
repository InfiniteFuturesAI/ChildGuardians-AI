# CHILD GUARDIANS — Project Charter

> **Version:** 1.0  
> **Status:** Draft  
> **Last Updated:** 2026-01-17

---

## 1. Purpose

CHILD GUARDIANS exists to protect children from exploitation by providing law enforcement with:

- **Evidence that survives court** — not just "leads"
- **Jurisdictional clarity** — not ambiguity
- **Procedural guardrails** — not training manuals
- **Global cooperation** — not bureaucratic friction
- **Oversight transparency** — not "trust us"

---

## 2. Core Identity

### 2.1 What This System IS

| Attribute | Description |
|-----------|-------------|
| **Passive** | Never interacts with criminals, never alters state |
| **Defensive** | Protects victims, preserves evidence, prevents errors |
| **Jurisdictional** | Respects sovereignty, routes evidence lawfully |
| **Evidentiary** | Every action is court-admissible by design |
| **Transparent** | All actions logged, auditable, overseen |
| **Pull-based** | Law enforcement queries; system doesn't push spam |

### 2.2 What This System is NOT

| Prohibition | Rationale |
|-------------|-----------|
| ❌ No deanonymization | Creates legal liability, breaks trust |
| ❌ No hacking | Violates law, poisons evidence |
| ❌ No sting operations | Outside system scope, requires human judgment |
| ❌ No suspect profiling | Creates civil liberties risk |
| ❌ No autonomous reporting | Requires threshold controls |
| ❌ No expansion beyond CSAM | Scope creep destroys trust |
| ❌ No content storage | Only hashes and metadata retained |
| ❌ No visual exposure | AI pre-filters before human review |

## 3. Non-Negotiable Design Assumptions

These are foundational truths the system is built upon. They cannot be overridden by policy, preference, or pressure.

### 3.1 Legal Reality

1. **Officers cannot violate local law to "get results"**  
   - System enforces jurisdiction, not officers

2. **Evidence must survive disclosure, defense challenge, and appeals**  
   - Defense Attorney Simulator validates before export

3. **Jurisdictional limits are real**  
   - Every evidence object contains a permission map

4. **International cooperation is slow**  
   - Design for asynchronous, treaty-aware routing

### 3.2 Human Reality

5. **Officers are overloaded, not malicious**  
   - Reduce burden; don't add training requirements

6. **Political pressure exists**  
   - Create process-based immunity from interference

7. **High-profile cases stall unless forced by process**  
   - Watchdog surfaces inactivity without accusations

### 3.3 System Reality

8. **If a mistake can invalidate a trial, the system must prevent the mistake from being possible**  
   - Not warn. Not recommend. Prevent.

9. **Evidence objects are append-only**  
   - History is never altered; corrections are recorded

10. **Transparency is non-optional**  
    - Every action logged, auditable, reportable

## 4. Stakeholders

### 4.1 Primary Users

| Stakeholder | Relationship |
|-------------|--------------|
| **National Cybercrime Units** | Query system, receive evidence objects |
| **International Bodies (Interpol, Europol)** | Route, coordinate, escalate |
| **Prosecutors** | Export court-ready evidence packages |
| **Victim Services** | Receive re-victimization alerts |

### 4.2 Oversight Bodies

| Stakeholder | Relationship |
|-------------|--------------|
| **Watchdog System** | Surfaces inactivity, access anomalies |
| **National Ombudsmen** | Receive aggregate transparency reports |
| **Public** | Sees annual transparency data (no PII) |

### 4.3 Excluded Parties

| Party | Reason |
|-------|--------|
| General public | No direct access to evidence |
| Private companies | No commercial access |
| Vigilante groups | Explicitly blocked |

---

## 5. Governance Principles

### 5.1 No Central Authority

- No single country commands the system
- Evidence lives globally; enforcement is local
- International bodies coordinate, not command

### 5.2 Sovereignty Preserved

- Each nation's laws are encoded in the jurisdiction map
- No country sees evidence it isn't entitled to
- Treaty requirements are enforced by design

### 5.3 Abuse Prevention

- All access logged
- Anomalies trigger watchdog
- Stalled cases are surfaced
- Political burial is visible

## 6. Success Criteria

| Metric | Target |
|--------|--------|
| **Evidence Admissibility Rate** | >95% pass Defense Simulator |
| **Jurisdictional Clarity** | 100% evidence objects have valid routing |
| **Procedural Error Rate** | <1% require remediation |
| **Reappearance Detection** | <24 hours from hash match to alert |
| **Victim Re-victimization Tracking** | 100% known victims monitored |
| **Watchdog Coverage** | 100% high-severity cases monitored |

## 7. Red Lines (System Will Never)

These are absolute prohibitions. Violating any of these is a system failure requiring immediate correction:

1. **Never store raw CSAM content**
2. **Never render illegal images to humans**
3. **Never train AI models on illegal content**
4. **Never interact with dark web forums (post, message, login)**
5. **Never deanonymize users**
6. **Never execute sting operations autonomously**
7. **Never profile suspects**
8. **Never expand scope beyond CSAM**
9. **Never delete evidence (only seal with justification)**
10. **Never operate without oversight**

---

## 8. Amendment Process

Changes to this Charter require:

1. Written proposal with rationale
2. Review by technical governance board
3. Review by legal advisory board
4. Public comment period (30 days)
5. Supermajority approval (>66%)
6. Version increment and publication

---

## 9. Founding Statement

This system exists because:

- Children deserve protection that doesn't depend on borders
- Officers deserve tools that don't create liability
- Prosecutors deserve evidence that survives court
- Victims deserve systems that don't re-traumatize
- The public deserves transparency without sensationalism

**This system does not create power. It removes places power can hide.**
